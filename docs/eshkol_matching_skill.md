# מיומנות התאמת רשויות אשכול (Eshkol Authority Matching Skill)

**מטרה:** הנחיה זו נועדה להנחות סוכני בינה מלאכותית (AI Agents) או מפתחים כיצד לשדך נכונה בין רשימת הרשויות והיישובים לבין כל בסיס נתונים אחר (למשל: למ"ס, ביטוח לאומי, משרד החינוך).
המיפויים הרשמיים והקודים שמורים בקובץ הצימודים המוכן: [טבלת_צימודים_למס.xlsx](טבלת_צימודים_למס.xlsx).

---

## 1. כלל ברזל: עצירה ובקשת אישור (Halt & Request Approval)
לפני ביצוע עיבוד נתונים או יצירת קבצי פלט (כמו Excel או JSON), חובה עליך (הסוכן):
1. לחלץ את כל שמות הרשויות או היישובים מקובץ הנתונים (ה-Dataset) המיועד.
2. לבצע הצלבה כפולה מול קובץ הצימודים המוכן [טבלת_צימודים_למס.xlsx](טבלת_צימודים_למס.xlsx).
3. **אם יש רשויות מקובץ המטרה שלא נמצאה להן התאמה ודאית במקור הנתונים** – עליך לעצור כל פעילות, להציג למשתמש את רשימת הרשויות החסרות, ולחכות להוראות ממנו.

---

## 2. חוקי התאמת שמות (Matching Rules)
אסור לבצע ניחושים עיוורים (Hallucinations). יש לפעול אך ורק לפי סדר הפעולות הבא:
1. **התאמה מדויקת (Exact Match)**: חפש התאמה של 1:1.
2. **ניקוי טקסט (Sanitization)**: נקה גרשיים, מירכאות, סוגריים ורווחים כפולים משני הצדדים, וחפש התאמה.
3. **מיפוי ידני קבוע (Hardcoded Mapping)**: השתמש במילון המיפויים הידועים מראש המפורט מטה.
4. **התאמה קרובה (Fuzzy Matching)**: שימוש באלגוריתם (כמו `difflib.get_close_matches`) מותר **אך ורק** אם סף ההתאמה (Cutoff) מוגדר ל-**85% לפחות**.

---

## 3. הנחיות לחילוץ ואקסטרקציה של נתונים מקובצי אקסל (Excel Data Extraction Guideline)
בכל פרויקט שבו נדרש לחלץ נתונים מקובץ אקסל חיצוני עבור ישויות אשכול, חובה לעמוד ב-2 תנאי סף:
1. **חילוץ שם הישות (טקסט) וגם קוד הישות (סמל למ"ס)**.
2. **אימות והצלבה של שניהם מול קובץ הצימודים המוכן** [טבלת_צימודים_למס.xlsx](טבלת_צימודים_למס.xlsx).
3. **אזהרת חפיפת סמלים (קריטי!)**: בקובצי למ"ס (כגון קובצי p_libud ואומדני אוכלוסייה), **סמלי מועצות אזוריות יכולים להיות זהים לחלוטין לסמלי יישובים** (למשל סמל 26 שייך למועצה אזורית "מטה יהודה" וגם ליישוב "ראש פינה"). לכן, חובה להצליב תמיד גם את הקוד וגם את השם. אם אין התאמה כפולה (או אם הסמל מתאים לרשות אבל השם לא), הנתון שגוי וחובה להתריע.
* **אם לא מצליחים לעמוד בתנאים אלו במלואם** (למשל, הקוד באקסל לא מתאים לשם, או שאחד מהם חסר), **חובה לכתוב זאת במפורש** בדו"ח הפלט/הלוגים של הריצה ולעצור לאישור המשתמש.

---

## 4. מודול צימודים מוכן בפייתון (Reusable Matching Script)
לנוחיותך, נכתב מודול פייתון רב-פעמי המיישם את כל חוקי הנרמול, המיפויים הידועים וההצלבה הכפולה (שם + קוד):
* **קובץ המודול:** [eshkol_matcher.py](eshkol_matcher.py)

**דוגמת שימוש בפייתון:**
```python
from eshkol_matcher import EshkolMatcher
import pandas as pd

# 1. אתחול הצימוד
matcher = EshkolMatcher()

# 2. טעינת קובץ האקסל החדש (למשל נתוני שכר)
df_salary = pd.read_excel("שכר_רשויות.xlsx")

# 3. הרצת הצלבה כפולה (שם עמודת טקסט, שם עמודת קוד)
df_result, errors = matcher.match_dataframe(df_salary, name_col="שם_רשות", code_col="קוד_רשות")

# 4. הדפסת שגיאות ואימותים לא תקינים במידה וישנם
if errors:
    print("Warning: Some records had verification mismatches!")
    for err in errors:
        print(err)
```
