# תיעוד ארכיטקטורה — דשבורד דמוגרפיה גליל מזרחי ומערבי

מסמך זה מתאר את הארכיטקטורה של המערכת לאחר השדרוג למבנה מונחה-תצורה (config-driven).

## 1. עקרון מנחה: מקור אמת יחיד
כל ההגדרה של "מי שייך לאשכול" נמצאת בקובץ אחד: **`data/config/eshkol_mapping.xlsx`**.
הוספת אשכול, רשות או יישוב = הוספת שורות לקובץ זה. **אין צורך בשינויי קוד.**

עמודות הקובץ (186 שורות):

| עמודה | תפקיד |
|---|---|
| שם ברשימת אשכול | שם הישות כפי שמוצג |
| סוג ישות | רשות / יישוב |
| קבוצה/שיוך | אשכול (לרשות) או שם המועצה האזורית (ליישוב) |
| קוד למס (סמל) | מפתח ההצטרפות לכל מקורות הלמ"ס |
| שם רשמי בלמס | שם קנוני לאימות מול המקור |
| שיטת התאמה | Exact / Manual / Fuzzy / Regional Council Custom Map |
| מגזר | יהודי / ערבי / דרוזי / מעורב / לא-יהודי |
| אשכול | גליל מזרחי / גליל מערבי |
| צורת יישוב | עירייה / מועצה מקומית / מועצה אזורית / קיבוץ / מושב / ... |

## 2. צינור הנתונים (ETL)
`etl/build_etl.py` טוען את קובץ התצורה ומאחד אליו את כל המקורות (לפי קוד למ"ס + אימות שם):

* **אומדני אוכלוסייה 2025** — אוכלוסייה עדכנית, ריבוי טבעי, מאזני הגירה. מועצות אזוריות מזוהות **לפי שם** (לא לפי קוד פנימי שביר).
* **גיל/מגדר 2023** — קריאה מאוחדת לשלושת הגיליונות (יהודים/לא-יהודים/מעורבים). בגיליון המעורב נשמרת רק קבוצת האוכלוסייה "סה"כ" כדי למנוע ספירה כפולה (הבאג שהפיל את הגליל המערבי).
* **חרדים 2023** · **p_libud 2024** (מדד סוציו-אקונומי, עולי 1990+, פריון, שטח, צפיפות).
* **מרשם אוכלוסין** (`residents_by_age_groups.csv`) — קבוצות גיל מצומצמות. בסיס מדידה רחב יותר מהאומדנים; משמש **רק** לשכבת הגיל המצומצמת ולמדד 0-18, ולא להחלפת נתון האוכלוסייה.

מועצות אזוריות אינן מופיעות בקובצי היישובים, ולכן הן מחושבות כסכום יישוביהן (לפי קובץ התצורה).

### שער ולידציה (Halt & Report)
לפני כתיבת `data/data.json` רצות בדיקות; **כל שגיאה עוצרת את הריצה** ולא נכתב פלט:
התאמת קוד⇄שם (חשש להתנגשות סמלים בלמ"ס), סכום גילאים ≤ אוכלוסייה (זיהוי ספירה כפולה),
מועצה אזורית = סכום יישוביה, כיסוי כל הקודים, והצלבת סך אשכול גליל מזרחי מול 196,456.

## 3. חישובים מרכזיים
* **גידול אוכלוסייה**: `(אוכלוסיית 2025 / אוכלוסיית 2023) − 1`.
* **יחס תלות**: `(0-14 + 65+) / (15-64)` על בסיס 2023.
* **0-18**: `גיל 0-5 + גיל 6-18` מתוך מרשם האוכלוסין (חישוב מדויק שאינו אפשרי מקובץ ה-5 שנים).
* אגרגציות אוכלוסייה הן סכומים; מדדים יחסיים מוצגים ברמת הישות (ללא ממוצע-של-ממוצעים).

## 4. הממשק (Frontend)
`etl/generate_dashboard.py` מייצר את `dashboard/` (HTML+JS עצמאי, מערכת העיצוב RKCG):
* KPIs: אוכלוסייה, מספר יישובים, 0-18 (מרשם), צעירים/עבודה/ותיקים.
* פירמידת גילאים עם מתג **מורחב/מצומצם** (התפלגות 5-שנתית 2023 ⇄ קבוצות גיל מרשם).
* רשת מדדים: סוציו-אקונומי, גידול, מאזן הגירה, עולי 1990+, גילאים, יחס תלות, פריון, צפיפות.
* סינון מדורג דינמי (אזור ← רשות ← מגזר ← צורה ← יישוב); מסנן האזור מתמלא מהנתונים.
* כלל מותג: אין אדום — מגנטה נושאת את הציר השלילי. הערות חישוב ומקור מתחת לכל גרף, והערת חיסיון (`..`).

## 5. הוספת אשכול חדש
לערוך את `data/config/eshkol_mapping.xlsx` (שורות + עמודות אשכול/מגזר/צורת יישוב), ולהריץ
`python etl/build_etl.py && python etl/generate_dashboard.py`. תווית האזור נגזרת אוטומטית משם האשכול.
